Activity 3.6 - Writing a subclass that overrides a superclass method

Topic

In this activity you will develop a subclass CommitteeMember that extends Member.

Materials

In addition to the Solution document, the project for this activity contains:

  • CommitteeMember, a class that is to extend Member but is empty so far, apart from comments!
  • Member, instances of which represent members of a local conservation group. This is the class used in Activities 3.4 and 3.5.
  • GroupRecord, an instance of which stores a list of Member objects. This is essentially the same class as the solution class in Activity 3.5
  • GroupRecordTest, which will test the CommitteeMember class that you write.

Task

You are to write the class CommitteeMember to extend Member, and add an extra instance variable position, which records the office a committee member holds, for example Secretary, Treasurer etc.

The class should have a four-argument constructor that sets all the instance variables, and a getter for the new attribute.

The toString method inherited from Member is to be overridden so that it will return a string including a committee member's position in addition to the details supplied for an ordinary member.

Instructions

Complete the code for the class CommitteeMember, as specified above, and run the project to test what you have written.
